AWS Glue のサービスにいき、サイドバーより クローラ を選択 →クローラの追加 なんのクローラか記名 (ex. event_id_master_crawler) 名前を記載 → Data stores → Crawl all folders データストア S3 右のフォルダマークより参照 →いいえ → 既存のIAMロールを選択 Default設定であれば AWSGlueServiceRoleDefault のはず 今のところ オンデマンドで実行 Glueで制作したデータベースを選択 (ex. shared-service) → 確認 先程作成したクローラを選択 → クローラの実行 完了後 サイドバーより テーブル でテーブルが作成されてるか確認できます。 csvファイルを指定するのではなく、そのファイルが⼊っているフォルダを選択 ステータスが Ready になるまで待ちます。 完了するのに数分かかります。
Amazon Athena に⾏き、サイドバーより クエリエディタ を選択 AwsDataCatalog データベース 作成したデータベースを選択 テーブル横の縦3つの点をクリック → テーブルのプレビュー 横にある 作成▼ をクリック → Create View Viewを作成するためのテンプレートが作成されます。 やりたい処理をここで編集します。 → 実行 問題がなければビューに追加されます。 画⾯右上ユーザーアイコンをクリック QuickSightの管理 セキュリティとアクセス権限 IAM role in use の中に使いたいリソースが⼊っているか確認 (今回はS3とAthenaを使いたいため、この2つがあることを確認) もしなければ Manage 追加したリソースを追加 → 保存 → 新しいデータセット Athenaを選択 選択すると先程作成したクローラのテーブルがテーブル欄に追加されます。 + をクリックすることで⼀つ⼀つのテーブルと型の確認が可能 クエリ結果でcsvファイルのデータが表⽰されればOK！ 1 CREATE OR REPLACE VIEW view_name AS 2 SELECT column1, column2 3 FROM table_name 4 WHERE condition; 権限の設定がされていないとデータセット追加できないので注意
